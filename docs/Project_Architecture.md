# 🐎 경마 데이터 파이프라인 프로젝트 구조 및 가이드

이 문서는 경마 데이터 분석 및 웹 표출을 위한 전체 파이프라인의 구조, 데이터 처리 단계, 그리고 최종 웹 애플리케이션 파일까지 프로젝트 전반을 이해하기 위해 작성된 가이드입니다.

---

## 📂 1. 전체 폴더 구조 (Directory Structure)

프로젝트는 데이터 흐름(Data Flow)과 소스 코드, 그리고 웹 출력부로 깔끔하게 나뉘어 있습니다.

```text
race_database/
├── data/                       # 파이프라인 데이터의 생명 주기를 담당
│   ├── 0_raw/                  # [입력] 원본 PDF, 텍스트 양식, 전문가 추천 등
│   ├── 1_intermediate/         # [과정] 파이프라인이 생성한 CSV 및 임시 TXT 파일
│   ├── 2_final/                # [최종] 웹에 연동할 최종 병합 JSON 데이터
│   └── 3_stats/                # [보조] 기수 및 조교사 통계 JSON
├── src/                        # 애플리케이션 핵심 로직 및 파이프라인
│   ├── RaceDataPipeline4.py    # ✨ 코어 구동 파이프라인 스크립트
│   ├── parsers/                # PDF/TXT 원본 데이터 추출 및 파싱 모듈
│   │   ├── kra_web_crawler.py  # 🤖 [핵심] KRA 홈페이지 데이터 수집 크롤러
│   │   └── RaceDataParser.py   # 📄 PDF 예상지 파싱 추출기
│   ├── converters/             # 포맷 변환 및 데이터 통합 모듈
│   ├── utils/                  # 기타 유틸리티 (통계 추출 등)
│   └── tests/                  # 테스트용 목업 및 스크립트
├── web/                        # 🌐 프론트엔드 (웹 애플리케이션)
│   ├── index.html              # 사용자가 보는 메인 웹 화면
│   └── firebase-config.js      # 파이어베이스 DB 연동 환경파일
├── docs/                       # 프로젝트 관련 문서
└── archive/                    # 더 이상 사용하지 않는 과거 코드 및 원본 보관소
```

## 🔄 2. 단계별 진행 과정 (Pipeline Flow)

데이터 파이프라인은 크롤링 대상 원본 수집(Phase 0)부터 시작하여 파싱 및 병합을 거쳐 정형화된 JSON 파일로 완성됩니다.

### [단계 0-1] KRA 웹 크롤러 데이터 수집 (`kra_web_crawler.py`)
- **작업:** Selenium을 이용해 한국마사회(KRA) 홈페이지에 접속, 해당 날짜의 경마 대상(출전표, 조교현황, 체중 등) 데이터를 자동 스크랩합니다. 
- **출력:** 프로젝트 루트 폴더에 임시로 `{지역}{MMDD}` 형태의 작업 폴더 생성 (예: `서울0315/`) 및 그 안에 CSV 파일 생성. 
*(참고: 생성된 임시 폴더들은 6단계 완료 후 자동 아카이브 처리됩니다)*

### [단계 0-2] PDF 원천 데이터 추출 (`RaceDataParser.py`)
- **입력:** `data/0_raw/s_run_hr_260215_all.pdf` (경마 잡지/출전표 PDF)
- **작업:** PDF를 읽어 각 경주별로 복승률, 승률, 상금, 초/후반 기록 등의 핵심 `DATA TIPS`를 추출.
- **출력:** `data/1_intermediate/{날짜}[지역]easyrace.txt`

### [단계 1] 출전표 파일 통합 (`integrate_entry_forms.py`)
- **입력:** 개별 출전표 데이터.
- **작업:** 각 경주에 출전하는 말의 기본 정보(마번, 마명, 기수, 조교사, 부담중량 등)를 하나의 파일로 병합.
- **출력:** `data/1_intermediate/{날짜}_{지역}_통합_출전표.csv`

### [단계 2] 10회 전적 상세 통합 (`Integration10raceRecord.py`)
- **입력:** 개별 말들의 최근 전적 데이터.
- **작업:** 출전하는 말들의 과거 10회 기록을 하나의 파일로 묶어서 분석 가능한 형태로 정제.
- **출력:** `data/1_intermediate/{날짜}_{지역}_경마_통합10회전적_상세.csv`

### [단계 3] JSON 형식을 위한 중간 데이터 변환 (`RaceDataPipeline` 내부 컨버터)
- **작업:** 단계 0~2에서 생성된 파일들과 전문가 추천(`data/0_raw/*ExpertPick.txt`) 파일을 모두 읽어들여, JSON 구조에 맞추기 위해 규격화된 CSV 중간 포맷으로 변환합니다.
- **출력 (전부 1_intermediate):** `EntrySheet.csv`, `RecentHistory.csv`, `RaceStats.csv`, `ExpertPick.csv`

### [단계 4] 심판 리포트/진료 현황 추가 결합 (`integrate_reports.py`)
- **작업:** 크롤러(`kra_web_crawler.py`)가 원본 데이터를 추출하면서 미리 생성해 둔 `_total.json` 파일에서 말들의 진료 이상 내역(`medical_alerts`) 및 심판 리포트(`steward_trip_notes`)를 읽어와, 파이프라인이 만들고 있는 JSON 객체의 각 출전마 데이터에 주입(Inject)합니다. 하위 폴더까지 탐색하여 데이터를 빈틈없이 병합합니다.
### [단계 5] 지역별 최종 JSON 생성 및 전체 병합 (`generate_json`, `MergeToEqualdateJson.py`)
- **작업:** 변환된 4개의 CSV를 하나로 조립하여, 웹에서 바로 읽을 수 있는 통합 구조의 JSON 파일로 압축 생성합니다. 만약 서울, 부산 지역이 모두 처리되었다면 두 지역을 하나의 파일로 합칩니다.
- **출력:** 
  - 지역별: `data/2_final/Seoul_Race_Data_{날짜}.json`, `Busan_Race_Data_{날짜}.json`
  - 전체 통합: `data/2_final/Merged_Race_Data_{날짜}.json` 

### [단계 6] 사용 완료된 원본 아카이브 (`shutil`)
- **작업:** 파이프라인이 성공하면 파싱이 끝난 날짜 폴더나 원본들을 안전하게 백업용 `archive/` 폴더로 이동시켜 `data/0_raw`를 항시 깨끗하게 유지합니다.

## 🎯 3. 최종 산출물 및 웹 파일 연동

파이프라인이 매끄럽게 종료되고 나면, 데이터는 프론트엔드 웹과 연동됩니다.

### 생성된 최종 파일 (`data/2_final/Merged_Race_Data_*.json`)
- **특징:** 특정 일자의 모든 경주(서울/부산 등)에 대해 **경주 기본 정보**, **출전마 상세 정보**, **과거 10전 기록**, **통계 분석**, **전문가 코멘트**가 모두 트리 형태로 기록되어 있습니다.

### 웹 파일 (`web/index.html`)
- **역할:** 사용자가 브라우저를 통해 접속하는 대시보드 화면입니다.
- **동작 방식:**
  1. `index.html` 내의 JavaScript가 `firebase-config.js`를 통해 DB(또는 로컬 JSON 파일)에 접근합니다.
  2. 파이프라인이 만들어낸 `Merged_Race_Data_*.json` 데이터를 읽어들입니다.
  3. 경주 탭 선택, 출전마 비교, 최근 전적 하이라이트 표시 등 직관적인 UI로 데이터를 렌더링(표출)합니다.
